9. Files in the src folder in an Angular project:
	i. index.html : This is what is rendered to the browser. This is the file that will be rendered to the browser.
			<app-root> means that components that are added to the body of index.html elsewhere in the app are 			rendered by <app-root></app-root>. <app-root></app-root> is what we call a selector. Here is the <app-			root> from this selector. app-root is app.html. The selector of app.html is app-root. After calling the 			app-root selector like this, the html code in app.html goes to <app-root></app-root> in index.html. It 			is called by <app-root></app-root> in index.html.

	ii. app.ts    : This is the root component of all components in the project.

	iii. app.module.ts : This is the module that has app.ts declared. That means app.ts is declared in this file.

		* Then the Angular framework uses a modular architecture. The concept of modules is widely used in JS 			  ( JavaScript ).

		* Latest versions (Angular 17+): New projects are "standalone" by default. Components, directives, and pipes 		  can import their own dependencies directly, removing the need for a central module file. The application is 		  bootstrapped using the bootstrapApplication function in main.ts, which takes the root standalone component 		  (AppComponent) and an application configuration (app.config.ts).

41. To Kill the Port:
	Open up cmd.exe (note: you may need to run it as an administrator, but this isn't always necessary), then run the below 		command:
			netstat -ano | findstr :<PORT>
			(Replace <PORT> with the port number you want, but keep the colon)
	The last column is the PID (process identifier). Locate the PID of the process that's using the port you want.
	Next, run the following command:
		taskkill /PID <PID> /F
		(No colon this time)
